The basics of Kataform

Kataform is an online app-based platform, coded in a programming language known as Go or Golang. You can visit the Go website at Golang.org.

About the Guide

This guide will be in parts and will have some interactive elements to allow you to practice the Go programming language. So, let's get started for you to learn the basics of Go!

Part One: The Basics of Go


Chapter One: The Header

Every Go file you code in will start the same way:



package main

import (

bunch of junk :)

)

That code will start the document, but most likely you will be using a template so you will never have to type that code. Under the import keyword, you will put all of the packages you want to import. Packages will allow you to import code, like a function that you can use to generate random numbers or print stuff out in your running app. Your imports will automatically be there in whatever template you use, but you should still know the mechanics of these.

You will also find that almost everything is in functions. This is something about Go and you will be coding the most within the mainrun function.

Oh, by the way a side note that somehow wormed its way out of a chapter: No semicolins in go, or spaces or tabs or anything else unless you want to code everything in one line.

Chapter Two: A Basic Intro to Variables and Functions

Functions:

There is nothing really different about functions in Go that you will need to know, except for the format of function creation (using functions is just functionName(parameters, more parameters, third parameter if needed). So, here is how to declare a function in Go:


func functionName(parameter names, more names, more) {
Bunch of Code
}

Variables:

You should already know about variables from prior coding, but there are some different rules in Go so here are ways to declare and use variables:

  • In any place, you can declare a variable: var variableName = blahblahblah
  • You should also put the type (I am using an example of the string type but there is also int, int32, int64, float32, float64 and a ton of other types): var variableName string = "My name is Kai and this is a string". Notice that I have the type of string after the variable name and before the equal sign.
  • Also, in functions only you can have a quicker variable creation setup: variableName := blahblah. Notice that you can NOT put a type in this way of creation.
  • To use a variable, just put the name of the variable.

Chapter Three: The Use of Printing

You will be running your apps in what is called a Terminal. You will type in something like ./App into the terminal and it will run your app. If you want to debug something or see what values are, in your code you can make it so it will print those things. Here is how to put printing in your code:

  • First, make sure that "fmt" is in the import section.
  • Second, in your code put: fmt.Printf(whatever you want to put)
  • Done!
